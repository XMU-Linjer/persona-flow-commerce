# PersonaFlow Commerce V1.0 项目架构

> 状态：还在设计中  
> 目标版本：V1.0.0  
> 内容：纯电商平台

---

## 1. 版本目标

V1.0 先完成一个可以正常运行的电商平台，跑通下面这条交易链路：

```text
用户登录
→ 浏览和搜索商品
→ 查看商品详情
→ 收藏或加入购物车
→ 提交订单
→ 扣减库存
→ 模拟支付
→ 查询订单
```

同时记录用户在商城里的主要操作：

```text
用户搜索、浏览、收藏、加购或购买
→ Java 业务模块产生行为事件
→ RabbitMQ 异步传递
→ 消费者持久化到 MySQL
→ Redis 更新搜索热词和实时兴趣统计
```

V1.0 主要完成：

- 完成常见的 Java Web 分层；
- 跑通商品、购物车、库存和订单之间的关系；
- 在实际功能中使用 MySQL、Redis 和 RabbitMQ；
- 为 V1.1 的用户行为分析和 Agent 接入准备数据与接口。

## 2. 总体架构

```text
┌─────────────────────────────────────────┐
│          Vue 3 + TypeScript Web         │
│                                         │
│ 登录 / 商品 / 搜索 / 收藏 / 购物车          │
│ 下单 / 支付 / 订单 / 简单后台               │
└────────────────────┬────────────────────┘
                     │ REST / JSON
                     ▼
┌──────────────────────────────────────────┐
│          Spring Boot 模块化单体            │
│                                          │
│ common      auth        user             │
│ product     search      favorite         │
│ cart        inventory   order            │
│ behavior    messaging   admin            │
└────────┬────────────┬─────────────┬──────┘
         │            │             │
         ▼            ▼             ▼
      MySQL         Redis        RabbitMQ
```

运行方式：

- 一个 Java 后端服务；
- 一个 Vue 前端；
- 一个 MySQL；
- 一个 Redis；
- 一个 RabbitMQ；
- 本地由 Docker Compose 管理基础设施。

## 6. Java 分层规范

每个业务模块按需要包含：

```text
controller
service
mapper
entity
dto
vo
```

调用方向：

```text
Controller
→ Service
→ Mapper
```

规则：

- Controller 接收请求并返回结果；
- Service 处理业务规则和事务；
- Mapper 负责数据库读写；
- Entity 对应数据库表；
- DTO 接收接口参数；
- VO 返回前端需要的数据；
- 模块之间通过 Service 调用；
- Controller 不直接调用 Mapper；
- 一个模块不直接调用另一个模块的 Mapper。

## 7. 核心业务模型

### 7.1 用户与权限

核心表：

```text
sys_user
sys_role
sys_user_role
user_address
```

### 7.2 商品

核心表：

```text
product_category
product_spu
product_sku
product_stock
```

关系：

```text
product_category 1 ── N product_spu
product_spu      1 ── N product_sku
product_sku      1 ── 1 product_stock
```

### 7.3 收藏与购物车

核心表：

```text
user_favorite
shopping_cart
```

唯一约束：

```text
user_favorite(user_id, spu_id)
shopping_cart(user_id, sku_id)
```

### 7.4 订单

核心表：

```text
customer_order
order_item
```

关系：

```text
sys_user       1 ── N customer_order
customer_order 1 ── N order_item
```

`order_item` 保存下单时快照：

- 商品名称；
- SKU 描述；
- 成交单价；
- 商品数量；
- 商品图片。

### 7.5 用户行为

核心表：

```text
user_behavior
search_record
mq_consume_log
```

每个行为事件至少包含：

```text
event_id
user_id
session_id
event_type
spu_id
sku_id
category_id
keyword
occurred_at
trace_id
context_json
```


## 9. Redis 职责

### 9.1 商品详情缓存

```text
commerce:product:detail:{spuId}
```

用于降低商品详情对 MySQL 的重复查询。

### 9.2 分类树缓存

```text
commerce:category:tree
```

用于缓存变化较少的商品分类结构。

### 9.3 搜索热词

```text
commerce:search:hot:{yyyyMMdd}
```

使用 Sorted Set 统计搜索词热度。

### 9.4 用户近期兴趣统计

```text
commerce:user:interest:{userId}
```

使用 Sorted Set 保存近期类别或标签分数。

建议初始行为权重：

```text
SEARCH            +2
VIEW              +1
FAVORITE          +4
ADD_TO_CART       +6
PURCHASE         +10
NEGATIVE_FEEDBACK -5
```

### 9.5 登录状态

```text
commerce:auth:blacklist:{tokenId}
```

用于主动退出后的 Token 失效控制。

## 10. RabbitMQ 职责

Exchange：

```text
commerce.behavior.exchange
```

类型：

```text
topic
```

Routing Key：

```text
behavior.search
behavior.view
behavior.favorite
behavior.cart
behavior.purchase
behavior.negative
```

队列：

```text
behavior.persist.queue
behavior.interest.queue
behavior.dead.queue
```

消息格式：

```json
{
  "eventId": "uuid",
  "eventType": "USER_SEARCHED",
  "source": "persona-commerce-server",
  "occurredAt": "2026-06-24T12:00:00Z",
  "traceId": "uuid",
  "version": "1.0",
  "payload": {}
}
```

消息链路：

```text
业务模块产生行为
→ Publisher 发送消息
→ persist.queue 保存行为
→ interest.queue 更新 Redis 统计
```

## 14. V1.0 完成标准

业务链路：

```text
用户登录
→ 搜索商品
→ 查看详情
→ 收藏或加入购物车
→ 创建订单
→ 扣减库存
→ 模拟支付
→ 查看订单
```

行为链路：

```text
业务行为产生
→ RabbitMQ
→ MySQL 存在行为记录
→ Redis 存在热词或兴趣分
```

完成后应满足：

- Spring Boot 可启动；
- Vue 可构建并运行；
- MySQL、Redis、RabbitMQ 连接正常；
- Flyway 可以初始化数据库；
- 核心接口有基本测试；
- 初始化数据可重复导入；
- Docker Compose 可以启动基础设施；
- README 更新为 V1.0 当前状态；
- Git 标签 `v1.0.0` 创建完成。
