# PersonaFlow Commerce V1.0 架构设计

> 状态：已完成  
> 目标版本：V1.0.0  
> 项目定位：可本地演示的纯电商平台主链路  
> 更新时间：2026-06-29

## 1. 文档目的

本文说明 V1.0 已完成的系统边界、模块职责、接口范围、数据表范围和基础设施使用方式。

V1.0 的目标是先完成一个清晰、可运行、可测试、可演示的电商平台主链路。行为事件流、RabbitMQ 业务事件、Agent 画像团队和 AI 购物洞察进入 V1.1 设计和实现。

## 2. V1.0 项目定位

V1.0 是一个 Spring Boot + Vue 的全栈电商 Demo，重点展示：

- account / catalog / shopping / trade 的模块化后端边界；
- 用户注册登录、商品浏览、收藏、购物车、地址、订单、库存和模拟支付；
- Vue 前端完整演示链路；
- MySQL / Redis / RabbitMQ 本地基础设施；
- Redis 商品详情缓存；
- 统一响应、错误码、JWT 鉴权、订单状态机和库存三段模型。

## 3. V1.0 已完成范围

### 3.1 后端模块

- account：注册、登录、JWT、当前用户、资料修改、密码修改、地址管理、`CurrentUserProvider`、`AddressQueryApi`。
- catalog：分类、SPU、SKU、商品列表、商品详情、SKU 查询、Redis 商品详情缓存、`ProductQueryApi`、`ProductSnapshot`。
- shopping：收藏 SKU、取消收藏、收藏列表、加购、修改数量、删除购物车项、清空购物车、购物车列表。
- trade：库存、创建订单、订单查询、取消订单、模拟支付、支付记录、库存锁定/释放/确认扣减。
- common：统一响应、统一错误码、业务异常、全局异常处理、分页结果等基础能力。

### 3.2 前端页面

- 登录、注册；
- 商品列表、商品详情；
- 收藏、购物车、地址管理；
- 结算页；
- 订单列表、订单详情、取消订单、模拟支付。

### 3.3 基础设施

- Docker Compose 管理 MySQL、Redis、RabbitMQ 容器；
- Flyway 管理 MySQL 表结构和演示数据；
- Redis 当前只用于商品详情缓存；
- RabbitMQ 容器在 V1.0 已准备为基础设施，但业务事件流进入 V1.1。

## 4. V1.0 不包含

V1.0 不实现以下能力：

- behavior 行为事件落库；
- RabbitMQ 行为事件发布和消费；
- RabbitMQ Agent 任务总线；
- Agent Team、用户画像、AI 购物洞察；
- Redis 搜索热词或兴趣统计；
- admin 管理接口；
- 真实支付、支付回调、退款、物流、优惠券、商家系统；
- 购物车结算跨模块 API。

## 6. 模块边界

| 模块 | Java 包 | V1.0 职责 |
|---|---|---|
| common | `common` | 统一响应、错误码、异常、分页、通用配置 |
| account | `auth`, `user` | 账号、认证、当前用户、地址、跨模块当前用户和地址快照 |
| catalog | `product` | 分类、SPU、SKU、商品浏览、商品详情缓存、商品快照 |
| shopping | `favorite`, `cart` | 收藏和购物车 |
| trade | `inventory`, `order`, `payment` | 库存、订单、模拟支付 |

V1.0 不创建 `search` 包。商品关键词搜索由 `product` 包内服务基于 MySQL LIKE 实现。

## 7. 跨模块协作

V1.0 使用稳定的跨模块 API，而不是直接访问其他模块的表、Mapper 或 Entity。

- shopping 通过 `CurrentUserProvider` 获取当前用户，通过 `ProductQueryApi` 校验和展示 SKU。
- trade 通过 `CurrentUserProvider` 获取当前用户，通过 `AddressQueryApi` 获取地址快照，通过 `ProductQueryApi` 获取商品快照。
- trade 不依赖 shopping，不从购物车创建订单，不在订单创建成功后清理购物车。
- catalog 不保存真实库存，不负责扣库存、锁库存或库存校验。

| 模块 | 表 |

| account | `sys_user`, `user_login_identity`, `sys_role`, `sys_user_role`, `user_address` |
| catalog | `product_category`, `product_spu`, `product_sku` |

V1.0 没有 `behavior_event`、`user_profile_version`、`search_record` 等行为和画像表。

## 10. 缓存与消息范围

Redis 当前只缓存商品详情：

```text
catalog:product:detail:{spuId}
TTL = 1 小时
```

V1.0 不实现搜索热词、用户兴趣统计或画像缓存。

RabbitMQ 容器已随 Docker Compose 准备，但 V1.0 不发布、不消费业务消息。`commerce.behavior.exchange` 和 `commerce.agent.exchange` 从 V1.1 开始设计和接入。

## 11. 安全规则

- `/api/auth/register` 匿名访问；
- `/api/auth/login` 匿名访问；
- `/api/catalog/**` 匿名访问；
- `/actuator/health` 匿名访问；
- `/api/admin/**` 需要 `ROLE_ADMIN`；
- `/api/shopping/**` 默认需要登录；
- `/api/trade/**` 默认需要登录；
- 其他接口默认需要认证。

## 12. 订单和库存状态

库存三段模型：

- 创建订单：`available_quantity -> locked_quantity`；
- 取消订单：`locked_quantity -> available_quantity`；
- 模拟支付成功：`locked_quantity -> sold_quantity`。

订单状态：

```text
10 = PENDING_PAYMENT
20 = PAID
30 = CANCELED
```

合法流转：

- `PENDING_PAYMENT -> PAID`
- `PENDING_PAYMENT -> CANCELED`

非法流转统一返回 `TRADE_ORDER_STATUS_NOT_ALLOWED`。

## 13. V1.1 衔接

V1.1 在 V1.0 主链路之上扩展：

- behavior 行为事件模块；
- RabbitMQ 行为事件总线；
- RabbitMQ Agent 任务总线；
- Python Profile Agent Team；
- 用户画像版本；
- 前端 AI 购物洞察展示。

V1.1 不改变 V1.0 已完成的 account、catalog、shopping、trade 主链路边界，只通过新增事件、表、服务和页面扩展能力。
